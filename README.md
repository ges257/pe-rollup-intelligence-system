## The Problem

Private Equity rollups face a **vendor consolidation challenge**: when acquiring multiple dental practices, which vendors should be standardized across the portfolio to maximize EBITDA? Traditional approaches rely on gut feel or simple cost comparisons.

**This system uses Graph Neural Networks to model the topology of vendor relationships and predict optimal consolidation paths.**

## Results

| Metric | Score | Target | Outcome |
|--------|-------|--------|---------|
| **PR-AUC** | **0.9407** | 0.937 (GBM baseline) | +0.37% over baseline |
| **Risk MAE** | **1.14 days** | < 3.0 days | Precise A/R forecasting |
| **Calibration (ECE)** | **0.0000** | < 0.10 | Perfect calibration |
| **Clustering (Silhouette)** | **0.3447** | > 0.30 | Valid pod structure |

> **Causal Validation:** Ablation studies confirm the model learned the underlying causal mechanism. Removing `Integration Quality` caused PR-AUC to drop by **-25.5%** (0.94 to 0.68), validating that the R-GCN correctly identified operational integration as the primary driver of vendor switching.

## Architecture

```
LLM Research → Synthetic Engine → R-GCN → Decision Dashboard
```

1. **Text-to-KG Pipeline**: Used Claude/GPT to extract 450 knowledge graph triples from 20 real vendor websites
2. **Causal Synthetic Data**: Engineered a mechanism-first data generator where integration quality drives switching probability
3. **R-GCN Model**: FastRGCNConv layers on heterogeneous graph (Sites ↔ Vendors with typed edges)
4. **Decision Support**: Streamlit dashboard translating predictions to enterprise value

See [ARCHITECTURE.md](ARCHITECTURE.md) for detailed system diagrams.

## Key Innovation: Causal Synthetic Data

Real PE data is fragmented, private, and sparse. Instead of random synthetic data, I built a **[Causal Synth Engine](https://github.com/ges257/causal-synth-engine)**:

```
Integration Quality → Acquisition Value → Vendor Switching
```

- **Integration Quality** (0/1/2) directly affects switching probability
- **Change Fatigue** reduces switching likelihood after recent transitions
- **Vendor Tier** affects KPI outcomes (Days A/R, Denial Rate)

The result: a dataset where the GNN must discover the causal relationships, not memorize arbitrary patterns.

See [causal-synth-engine](https://github.com/ges257/causal-synth-engine) for the full data generation pipeline.
